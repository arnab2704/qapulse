How Allure generates results
When you run tests with the Allure listener, it writes one UUID.json file per test into an allure-results/ folder. This tab accepts that folder as a ZIP archive or lets you upload individual UUID.json files directly.

Multi-region / multi-device ZIPs — if your ZIP contains sub-folders (one per region or device), QA Pulse creates a separate run for each sub-folder automatically.
ACCEPTED ZIP STRUCTURES
Option A — single folder:
  📦 allure_results.zip
    └── allure-results/
        ├── 8f14e45f-…uuid….json  ← test result
        ├── 9b2e4a1c-…uuid….json
        ├── environment.properties  ← auto-read
        ├── executor.json  ← CI build info
        └── *.png / *.txt  ← ignored
Option B — multi-region (one run per sub-folder):
  📦 all_regions.zip
    ├── UK_desktop-chrome/    → run 1
    │   ├── uuid1.json, uuid2.json …
    ├── DE_android-chrome/    → run 2
    └── AT_iphone-safari/     → run 3
Option C — flat ZIP (no sub-folder):
  📦 results.zip                → treated as one run
    ├── uuid1.json, uuid2.json …
    └── environment.properties

📂 JSON Data Store
✓ Plain JSON — no database needed
All data is stored as human-readable JSON files. Open them in VS Code, Notepad, or any text editor. Copy the data/ folder to back up everything.
File structure
data/
  runs.json           ← all run metadata (edit freely)
  tests/
    <run-id>.json   ← test cases for each run
